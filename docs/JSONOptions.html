JSONOptions
Class

Description

These are options for JSON serialization and reflection data representation. Note that, even though you can specify the header path, type, and parse function for a json type, the only JSON library really supported is nlohmann-json.

Fields

bool Use true Whether or not to use JSON C++ types to represent attribute reflection data (instead of just a stringified version).
string HeaderPath "<nlohmann/json.hpp>" The header path to include in order for JSON attribute data to work
string Type "::nlohmann::json" The type that stores the JSON attribute data
string ParseFunction "::nlohmann::json::parse" The function that takes the stringified attribute data and turns into the JSON type
bool AlwaysSaveAllFields false When converting object fields to JSON fields, the serializer will check if the field is equal to its initializer, skipping serializing that field if it is equal. Set AlwaysSaveAllFields to true to always save all object fields to JSON.
bool GenerateSerializationMethods true Toggles generation of JSON serialization methods for reflected classes
string ObjectTypeFieldName "$type" The name of the field that holds the type name of the object stored when storing a polymorphic object
string ObjectGUIDFieldName "$guid" The name of the field that holds the GUID of the type of the object stored when storing a polymorphic object
bool AllowSchemaGenerationPerClass true TODO: This
bool IgnoreInvalidObjectFields false If true, errors while deserializing non-required object fields will be silently ignored, the fields will be reset, and deserialization will continue. If false, errors (exceptions) will be propagated.
